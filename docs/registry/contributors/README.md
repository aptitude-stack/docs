# Contributor Docs

Use this directory when you need to work in the repo rather than just understand the product.
These docs are the tutorial and how-to layer for engineers, reviewers, and operators.

## What Belongs Here

- local setup and common run commands
- testing and verification workflow
- module ownership and contributor navigation
- rules for maintaining canonical docs

## What Does Not

- live route or schema reference
- architecture rationale or boundary decisions
- forward-looking design proposals or milestone history
